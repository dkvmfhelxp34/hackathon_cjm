# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
제안서(용역·R&D) **공고문/제안요청서(RFP)를 사람이 읽고 핵심 항목(예산·기간·평가배점·제출서류 등)을 손으로 표/대시보드에 옮기던 작업**을, agent1(수집·파싱)→agent2(AI 분석, 본인)→agent3(시각화) 파이프라인으로 자동화. 본인(마경림)은 **agent2 = AI 분석 엔진** 담당.

## Before (기존 수작업 절차)
1. 공고문(HWP/PDF)을 사람이 처음부터 끝까지 정독.
2. 사업명·발주처·예산·기간·평가배점·제출서류·마감일을 눈으로 찾아 손으로 표에 옮김.
3. 평가배점 합계가 100인지 직접 계산해 검산, 제출서류 누락 여부를 수동 점검.
4. 근거(몇 쪽에 있었는지)는 따로 기록하지 않아 추후 재확인 시 다시 문서를 뒤짐.
5. 문서마다 정리 양식이 조금씩 달라져 대시보드로 합칠 때 형식이 깨짐.

## After (에이전트화된 절차)
1. agent1이 파싱한 결과(JSON)를 agent2가 입력으로 받음(입력 계약 mock으로 고정).
2. agent2가 Claude **tool use(고정 JSON 스키마 강제)** 로 예산·기간·평가배점·제출서류를 구조화 추출.
3. 스키마 검증 → 평가배점 합계 자동 검산 → 모든 항목에 **근거 페이지/evidence_id 자동 연결**.
4. 결과를 agent3 대시보드가 그대로 소비하는 **일관된 JSON 계약**으로 산출.

## 효과 (자기 업무에 맞는 지표로 — 형식 자유)
> 측정 시점: 1건짜리 mock 공고문으로 프로토타입 end-to-end 검증한 결과 기준. (정확한 Before 소요시간은 실제 업무 기준 추후 보정)

| 지표(자기 업무에 맞게) | Before | After | 효과 |
|------|--------|-------|------|
| 형식 일관성 | 문서마다 양식 제각각 → 합칠 때 깨짐 | 고정 JSON 스키마 100% 준수(4종 도구 PASS) | 대시보드 병합 시 형식 오류 0 |
| 근거 추적 | 별도 기록 안 함 → 재확인 시 재검색 | 전 항목 evidence_id 자동 연결 | 클릭 한 번으로 원문 근거 추적 |
| 평가배점 검산 | 사람이 수동 합산 | 합계 자동 검증(불일치 시 플래그) | 검산 누락/오류 방지 |
| 할루시네이션 | 해당 없음(사람이 직접) | 추정 금지·근거 없으면 미생성 강제 | 없는 값 생성 0 |
| 처리 방식 | 1건씩 수동 정독·타이핑 | 입력만 주면 자동 추출(건수 무관) | 문서 늘어도 동일 절차 반복 |
