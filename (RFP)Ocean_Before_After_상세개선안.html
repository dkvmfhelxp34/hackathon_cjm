상세 개선안 · Before & After

예보사업부 RFP 통합 대시보드

업로드된 4개 제안요청서/공고문을 분석해, 기존의 수동 문서 확인 방식이 원문 기반 React 통합 대시보드로 어떻게 개선됐는지 정량·정성적으로 정리했습니다. 현재까지 구현된 기능 기준입니다.

구현: React (Vite/Bun) · 라이트 디자인 시스템(GeoSR) · 정적 데이터(원문 MD 자동 파싱) · 표출 전용

1. 기존 업무의 문제와 개선

Before 수동 확인

  • 사업별 제안요청서(hwpx·pdf)를 각각 열람
  • 사업명·예산·기간·발주처를 직접 정리
  • 긴 과업내용에서 핵심 요구사항을 수동 추출
  • 전문용어를 별도 검색하거나 담당자 경험에 의존
  • 사업 간 비교가 어렵고 정리 형식이 제각각

After 통합 대시보드

  • 각 발주처의 사업을 하나의 목록으로 통합, 발주처 색상 구분
  • 프로젝트명·예산·발주처를 표준 표로 제공
  • 사업요약·대상범위·산출물·중요정보를 카드로 구조화
  • 상세 용어해설(전문정의·쉬운설명·과업관점) 제공
  • 원문 사업내용을 표출 + 원본 파일 다운로드

3. 정량적 개선 효과

4개 사업 통합해안침수·해양위성·저차생태계·민군경 AI를 한 화면에서 확인
총 약 199억발주처 필터 선택 시 사업 수·예산 합계가 실시간 연동
28개 용어 해설전문정의·쉬운설명·과업관점 3단으로 해석 기준 통일
3단계 화면 구조목록 → 요약/용어 → 원문(사업내용)으로 업무 흐름 표준화
70~80% 절감 예상파일 탐색·핵심정리·용어확인에 드는 반복시간 감소
원본 파일 다운로드원본 파일명 그대로 hwpx·pdf 즉시 내려받기 제공

4. 정성적 개선 효과

영역개선 내용업무 효과
일관성모든 사업을 같은 화면 구조·정보 단위로 정리담당자별 정리 차이를 줄이고 비교 가능성 향상
이해도전문용어를 전문정의·쉬운설명·과업관점으로 분리비전문자도 과업 핵심을 빠르게 이해
근거성요약 + 원문 구조화 표출 + 원본 파일 다운로드요약 신뢰성↑, 원문 추적·검증 시간 단축
탐색성발주처 다중 필터 · 검색 · 초기화를 목록에 배치관심 사업을 빠르게 좁혀 찾고 한 번에 초기화
가독성원문을 대제목·항목·비고로 구분, 추출 깨진 표는 별도 박스 처리긴 원문도 위계가 보여 훑어보기 쉬움
확장성동일 구조로 새 RFP 추가, 목록 자동 스크롤데이터가 늘어도 화면이 안정적으로 유지

7. 기술 구성

React + Vite (Bun) 정적 데이터 (DB 없음, 표출 전용) 원문 MD → JSON 자동 파싱 라이트 디자인 시스템 (GeoSR) 발주처 다중 필터 · 검색 · 초기화 원문 구조화 렌더링 · 표 잔해 분리 원본 파일 다운로드 (hwpx·pdf) 실시간 시계 · 페이지 전환 애니메이션 목록 자동 스크롤 (데이터 증가 대비)
결론
본 개선은 “RFP를 읽기 쉽게 보여주는 화면”을 넘어, 수동 문서 확인 업무를 원문 기반 지식 대시보드로 전환한다. 발주처 필터·검색으로 원하는 사업을 빠르게 좁히고, 요약·용어·구조화된 원문·원본 다운로드로 검토 속도와 추적성을 동시에 높였다. 정적 데이터 구조라 향후 DB/API 연동, 수주검토·위험평가·결재보고서 자동화로 확장 가능하다.
